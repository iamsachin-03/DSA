Breadth First Search

Explore a graph layer by layer — every neighbor before going deeper. The queue is the soul of BFS.

🧠 Core Intuition
The Ripple Analogy Drop a stone in a pond. Waves spread outward in concentric rings — that's BFS. Visit all nodes at distance 1, then distance 2, then 3…
Real-World Parallel Friend-of-a-friend discovery on social media. Level 1 = direct friends. Level 2 = friends of friends. BFS finds shortest paths.
The Secret Weapon Mark visited WHEN ENQUEUING, not when dequeuing. This stops duplicate entries in the queue — a classic bug.
⚡ Complexity
⏱ TIME: O(V + E)

Each vertex dequeued once O(V). Each adjacency list edge examined once O(E).

💾 SPACE: O(V + E)

Visited array O(V) + Queue max size O(V) + Adjacency list O(V+E).

🧩 Algorithm Logic Deep Dive
Why Queue and not Stack? A queue gives FIFO order — process nodes in the order they were discovered. A stack gives LIFO (DFS). The data structure IS the algorithm.
Why Mark Visited on Enqueue? If you mark on dequeue, the same node can enter the queue multiple times before being processed. This causes redundant work and wrong output.
Level-by-Level Property All nodes at distance d from source are visited before any node at distance d+1. This guarantees BFS finds shortest paths in unweighted graphs.
Adjacency List vs Matrix With adj list: O(V+E) time. With adj matrix: O(V²) time even for sparse graphs. Always prefer adj list for BFS.

Print Adjacency List

Given V vertices and E edges, build the adjacency list representation of an undirected graph. For every edge (u,v) — add v to u's list AND u to v's list.

🧠 Core Intuition
The Phone Book Analogy Imagine a phone book where each person lists all their direct contacts. An adjacency list is exactly that — for each node, a list of everyone it's directly connected to.
Undirected = Two-Way Street Edge (u, v) means u can reach v AND v can reach u. So every edge adds TWO entries — v into adj[u] AND u into adj[v]. Forget either direction → wrong answer.
Why Not Adjacency Matrix? Matrix uses O(V²) space even for sparse graphs. Adjacency list uses O(V+E) — far more memory-efficient when E << V².
⚡ Complexity
⏱ TIME: O(V + E)

Initialize V empty lists O(V). Process each of E edges twice O(2E) = O(E).

💾 SPACE: O(V + E)

V lists for vertices + 2E entries total across all lists (undirected stores each edge twice).

UNDIRECTED GRAPH REPR FOUNDATION
Common Gotcha Output order matters! The problem says sort neighbors. Always sort each adjacency list before returning.

import java.util.*; class Solution { public List<List<Integer>> printGraph( int V, int[][] edges) { // Step 1: Init V empty ArrayLists List<List<Integer>> adj = new ArrayList<>(); for (int i = 0; i < V; i++) adj.add(new ArrayList<>()); // Step 2: Process each edge — both ways for (int[] edge : edges) { int u = edge[0], v = edge[1]; adj.get(u).add(v); // u → v adj.get(v).add(u); // v → u } // Step 3: Sort each adjacency list for (int i = 0; i < V; i++) Collections.sort(adj.get(i)); return adj; } // Test public static void main(String[] args) { Solution sol = new Solution(); int[][] edges = {{0,1},{0,4},{4,1}, {4,3},{1,3},{1,2},{3,2}}; System.out.println(sol.printGraph(5, edges)); // [[1,4],[0,2,3,4],[1,3],[1,2,4],[0,1,3]] } }

①
Every undirected edge = 2 inserts. adj[u].append(v) AND adj[v].append(u). Missing one makes it directed — wrong.
②
Adjacency list = foundation of ALL graph algos. BFS, DFS, Dijkstra, Prims, Kruskal — all start from adj list.
③
Space O(V+E), not O(V²). For a graph with 1000 nodes and 2000 edges: list needs 2000 entries, matrix needs 1,000,000.
④
Sort requirement is problem-specific. Most graph problems don't require sorted neighbors. Only sort when asked.
⑤
0-based indexing. Array indexed 0 to V-1. Node 0 = adj[0]. Never create V+1 size by mistake.
Mental Model to Never Forget Think of adj[i] as "the friend list of person i". Building adjacency list = exchanging phone numbers — when A adds B, B must also add A. Symmetric, always.

Depth First Search

Go as deep as possible before backtracking. DFS uses the call stack (recursion) or an explicit stack — it dives into one branch fully before exploring siblings.

🧠 Core Intuition
The Maze Explorer Analogy Imagine exploring a maze — always take the first available turn, keep going until you hit a dead end, then backtrack to the last junction and try the next turn. That's DFS.
Stack = DFS (Recursion = Hidden Stack) Recursion uses the call stack implicitly. When you call dfs(neighbor), you push a frame. When it returns, you pop — that IS backtracking. The stack IS the algorithm.
BFS vs DFS — The Key Difference BFS uses a Queue → explores level by level (width first). DFS uses a Stack → explores branch fully (depth first). Same visited[] trick, different data structure = totally different traversal.
⚡ Complexity
⏱ TIME: O(V + E)

Each vertex visited once O(V). Each edge examined once per endpoint O(E). Total: O(V+E).

💾 SPACE: O(V + E)

Visited array O(V) + Recursion stack depth up to O(V) + Adjacency list O(V+E).

RECURSIVE BACKTRACKING STACK-BASED
Stack Overflow Warning For V up to 10⁴, recursion depth can reach 10⁴. Python's default recursion limit is 1000! Use sys.setrecursionlimit() or iterative DFS for large inputs.

def dfsOfGraph(adj): n = len(adj) visited = [False] * n result = [] def dfs(node): # Mark visited IMMEDIATELY on entry visited[node] = True result.append(node) # Recurse into neighbors left-to-right for neighbor in adj[node]: if not visited[neighbor]: dfs(neighbor) # ← dive deep # implicit return = backtrack dfs(0) # Start from vertex 0 return result # ── Iterative version (avoids stack overflow) ── def dfsIterative(adj): n = len(adj) visited = [False] * n result = [] stack = [0] visited[0] = True while stack: node = stack.pop() # LIFO! result.append(node) for nb in reversed(adj[node]): if not visited[nb]: visited[nb] = True stack.append(nb) return result # Test adj = [[2,3,1],[0],[0,4],[0],[2]] print(dfsOfGraph(adj)) # [0, 2, 4, 3, 1]
import java.util.*; class Solution { private List<Integer> result; private boolean[] visited; public List<Integer> dfsOfGraph( List<List<Integer>> adj) { int n = adj.size(); result = new ArrayList<>(); visited = new boolean[n]; dfs(0, adj); // start from 0 return result; } private void dfs(int node, List<List<Integer>> adj) { // Mark visited on entry visited[node] = true; result.add(node); // Explore each neighbor left-to-right for (int neighbor : adj.get(node)) { if (!visited[neighbor]) { dfs(neighbor, adj); // recurse } } // function returns → backtrack } /* ── Iterative DFS using Stack ── */ public List<Integer> dfsIterative( List<List<Integer>> adj) { int n = adj.size(); boolean[] vis = new boolean[n]; List<Integer> res = new ArrayList<>(); Deque<Integer> stack = new ArrayDeque<>(); stack.push(0); vis[0] = true; while (!stack.isEmpty()) { int node = stack.pop(); res.add(node); List<Integer> nbrs = adj.get(node); for (int i = nbrs.size()-1; i >= 0; i--) { if (!vis[nbrs.get(i)]) { vis[nbrs.get(i)] = true; stack.push(nbrs.get(i)); } } } return res; } }

Backtracking is Just Returning When dfs(4) returns → control goes back to dfs(2). dfs(2) has no more unvisited neighbors → returns to dfs(0). dfs(0) then visits 3, then 1. Each return IS the backtrack step.
Iterative: reversed() is Critical When using an explicit stack, push neighbors in REVERSE order so the first neighbor (left) gets popped first. Without reversal, you'd get right-to-left order.
🎯 Key Takeaways
①Stack = DFS. Recursion is an implicit stack. Explicit stack gives iterative DFS.
②Mark visited on ENTRY. Before exploring neighbors, not after. Same rule as BFS.
③Backtrack = function return. No explicit undo needed for visited[] in connected undirected graph traversal.
④Left-to-right order = iterate adj[node] without reversing (recursive). Iterative: push reversed.
⑤Python: set recursion limit for large V, or use iterative DFS to avoid RecursionError.

Number of Provinces

LeetCode 547 · Medium — Count connected components in an undirected graph given as an adjacency matrix. Each connected component = one province.

🧠 Core Intuition
The Island Counting Analogy Imagine cities as dots on a map. Draw lines between connected cities. Every isolated cluster of connected dots = one province. Your job: count how many clusters exist.
The Key Insight Start DFS/BFS from any unvisited city — it will visit ALL cities in its province. Then increment province count. Repeat from the next unvisited city. Each "fresh start" = a new province.
Matrix → Graph Translation isConnected[i][j] = 1 means edge between city i and city j. The diagonal isConnected[i][i] is always 1 (a city is connected to itself) — skip it when iterating neighbors.
⚡ Complexity
⏱ TIME: O(N²)

We visit each cell in the N×N matrix once. DFS from every city: total work = O(N²) since each row has N entries to check.

💾 SPACE: O(N)

Visited array of size N + recursion stack depth up to O(N) in worst case (linear chain of cities).

CONNECTED COMPONENTS DFS / BFS / UNION-FIND
3 Valid Approaches DFS, BFS, or Union-Find all solve this. DFS is simplest to code. Union-Find is most elegant for dynamic graphs. All share O(N²) time here because of the matrix.

class Solution { public int findCircleNum(int[][] isConnected) { int n = isConnected.length; boolean[] visited = new boolean[n]; int provinces = 0; for (int i = 0; i < n; i++) { if (!visited[i]) { provinces++; // new province! dfs(isConnected, visited, i, n); } } return provinces; } private void dfs(int[][] grid, boolean[] visited, int city, int n) { visited[city] = true; for (int nb = 0; nb < n; nb++) { if (grid[city][nb] == 1 && !visited[nb]) { dfs(grid, visited, nb, n); } } } // Time: O(N²) Space: O(N) }
🎯 Key Takeaways
①Province = Connected Component. Count how many times you start a fresh DFS.
②Outer loop is critical. for i in range(n): if not visited[i] → provinces++. This is the counter driver.
③Matrix vs Adj List DFS. Instead of "for nb in adj[node]", you do "for nb in range(n): if matrix[node][nb]==1".
④Diagonal skip is automatic. isConnected[i][i]=1 but visited[i]=True already so it's skipped naturally.
⑤Union-Find also works and is preferred in dynamic/online settings. For static matrix DFS is cleaner.

    dfs(nb)

Number of Islands

LeetCode 200 · Medium — Count connected components of '1's in a 2D grid. Each group of connected land cells (4-directionally) = one island. Classic flood-fill with DFS/BFS.

🧠 Core Intuition
Flood Fill / Paint Bucket Analogy Imagine the Photoshop paint bucket tool — click on any land cell, it floods and colors the entire connected island. Count how many times you have to click = number of islands.
The Key Pattern — Same as Provinces Outer loop: scan every cell. If it's '1' (land) AND unvisited → new island found! Run DFS/BFS to sink the entire island (mark all connected '1's as visited). Repeat.
4-Directional, Not 8 Only UP/DOWN/LEFT/RIGHT count as connected. Diagonal neighbors do NOT form the same island. The 4 directions = [(-1,0),(1,0),(0,-1),(0,1)].
⚡ Complexity
⏱ TIME: O(M × N)

Every cell is visited at most twice — once by the outer loop, once by DFS. M = rows, N = cols. Total work = O(M·N).

💾 SPACE: O(M × N)

Visited array O(M·N) + DFS recursion stack can go up to O(M·N) in worst case (one giant island snaking through the grid).

In-place Trick You can avoid a visited array entirely by setting grid[r][c]='0' (sink the island) when you visit a cell. This saves O(M·N) space but mutates the input.

Rotting Oranges

LeetCode 994 · Medium — The key insight: all rotten oranges rot simultaneously each minute. This is Multi-Source BFS — seed the queue with ALL rotten oranges at once, then expand level by level. Each BFS level = 1 minute.

🧠 Core Intuition
Multi-Source BFS — The Key Pattern Unlike single-source BFS (one start node), here ALL rotten oranges are sources simultaneously. Enqueue every '2' at minute 0. Each BFS "level" expansion = 1 minute of rotting. BFS level-count = answer.
Why BFS, Not DFS? DFS would give the time from one rotten orange — not the simultaneous spread. BFS naturally models "everything at distance d happens before distance d+1". The minutes elapsed = the BFS depth when all fresh oranges are consumed.
The -1 Case After BFS finishes, if any fresh orange ('1') still remains in the grid, it was unreachable from any rotten source (isolated by walls/gaps). Return -1. Otherwise return the minutes count.
⚡ Complexity
⏱ TIME: O(M × N)

Every cell is enqueued and dequeued at most once. Initial scan for fresh/rotten counts is O(M·N). Total: O(M·N).

💾 SPACE: O(M × N)

BFS queue can hold up to O(M·N) cells in worst case (all cells rotten at start). Grid modification avoids extra visited array.

This is a BFS-ONLY problem DFS cannot correctly compute "minimum time" here. The simultaneous spread model requires BFS level-by-level processing. This is a canonical Multi-Source BFS problem.

Undirected Graph Cycle

GFG · Medium — Detect if an undirected graph contains a cycle. The trick: in BFS/DFS, if you encounter a visited neighbor that is NOT your direct parent, you've found a back edge → cycle exists.

🧠 Core Intuition
The Parent Trick — The Heart of the Algorithm In an undirected graph, every edge is bidirectional. When DFS/BFS visits node A via node B, B is A's parent. If A then sees B as a neighbor, that's just the same edge back — NOT a cycle. A cycle only exists if A sees another visited node that is NOT its parent.
Multi-Component Graphs The graph may have multiple disconnected components. Run BFS/DFS from every unvisited node. If ANY component has a cycle, return true. One outer loop over all vertices covers this.
DFS Back Edge = Cycle In DFS: if you reach a visited node that isn't your parent → back edge → cycle. In BFS: if you find a visited neighbor ≠ parent in the queue → cycle. Both approaches work in O(V+E).
⚡ Complexity
⏱ TIME: O(V + E)

Each vertex and edge visited at most once across all BFS/DFS calls. Building the adjacency list: O(E). Total: O(V+E).

💾 SPACE: O(V)

Visited array O(V) + BFS queue or DFS stack O(V) + adjacency list O(V+E). Auxiliary space (excluding adjacency list) is O(V).

BACK EDGE PARENT TRACKING MULTI-COMPONENT
Union-Find Also Works For each edge (u,v): if u and v are already in the same set → cycle detected. Union-Find is often preferred in interview settings for its elegance. Both are O(V+E).

🧩 The Parent Check — Visual Proof
The Rule in One Line visited[nb] == true AND nb != parent → back edge → cycle exists. This is the entire algorithm condensed.

Parallel edges edge case If there are multiple edges between the same pair (u,v), using parent ID isn't enough — node v could have two edges to u, so seeing u as visited ≠ cycling. Track parent edge index instead, or use Union-Find which handles this naturally.

LeetCode 542 · Medium — Find the distance to the nearest 0 for every cell. The key insight: run Multi-Source BFS from all 0-cells simultaneously — they are the sources, 1-cells are the destinations. This is the inverse of Rotting Oranges.

🧠 Core Intuition
Reverse Thinking — The Key Insight "Distance from each 1 to the nearest 0" sounds like running BFS from every 1. But that's O(M·N·(M+N)) — too slow! Instead, reverse it: seed ALL 0-cells at distance 0 and BFS outward. Every 1-cell gets assigned a distance when first reached. O(M·N) total.
Multi-Source BFS Pattern (Same as Rotting Oranges) Enqueue all zeros upfront at distance 0. BFS expands one level per unit distance. The first time BFS reaches a 1-cell, that distance IS the shortest distance to any zero (BFS guarantees shortest path in unweighted graphs).
Why Not DP? DP works too: pass top-left to bottom-right, then bottom-right to top-left. O(M·N) time, O(1) extra space. But Multi-Source BFS is the canonical interview answer — it generalises to any grid distance problem.
⚡ Complexity
⏱ TIME: O(M × N)

Each cell is enqueued and processed exactly once. Two passes: scan for 0s O(M·N) + BFS O(M·N).

💾 SPACE: O(M × N)

Output distance matrix O(M·N) + BFS queue O(M·N) worst case (all cells are 0).

MULTI-SOURCE BFS SHORTEST PATH REVERSE THINKING
The Wrong Approach to Recognise Running BFS from each 1-cell separately = O((M·N)²) — TLE on large grids. The insight of seeding from 0-cells is what separates an optimal solution from a naive one.

🔑 Reverse Thinking — Why Seed from 0s, Not 1s
❌ NAIVE: BFS from each 1
For every 1-cell, run a full BFS to find its nearest 0.
If there are K ones → K BFS runs.
Worst case: O(M·N·(M+N)) — way too slow.
✓ OPTIMAL: BFS from all 0s
Seed all 0-cells at dist=0.
One BFS expansion fills everything.
O(M·N) total — 10,000× faster.
Connection to Rotting Oranges (LC 994) Identical pattern: seed multiple sources → BFS out → record "time/distance" when each cell is first reached. In Rotting Oranges, sources=rotten cells, destination=all fresh cells. Here, sources=all 0s, destination=all 1s. One algorithm, two problems.

🎯 Key Takeaways
①Seed from 0s, not 1s. The direction reversal is the entire insight. O(M·N) vs O(M²·N²).
②dist > d+1 is your visited check. If dist[nr][nc] is already ≤ d+1, don't enqueue. No separate boolean array needed.
③BFS = shortest path guarantee. The first time BFS reaches a cell, that's the shortest distance. Never need to update twice.
④DP alternative: O(1) space. Two-pass DP avoids BFS queue entirely. Top-left→BR then BR→top-left. Both are O(M·N) time.
⑤Same pattern as Rotting Oranges. Multi-source BFS with (r,c,dist) tuple. The last dist seen = answer for Oranges; here every dist matters.

Surrounded Regions

LeetCode 130 · Medium — Capture all 'O' regions surrounded by 'X'. The trick: think in reverse. Instead of finding surrounded O's, find the SAFE O's (those touching the border) and mark them. Everything else gets captured.

🧠 Core Intuition — The 3-Phase Algorithm
PHASE 1 — Mark Safe O's
BFS/DFS from every 'O' on the board border. Mark all connected O's as 'S' (Safe). These can NEVER be captured — they touch the edge.
PHASE 2 — Capture Interior O's
Scan the whole board. Any remaining 'O' (not marked 'S') is completely surrounded → flip to 'X'. These are the captured regions.
PHASE 3 — Restore Safe O's
Flip all 'S' cells back to 'O'. The board is now in its final state with only surrounded regions captured.
⚡ Complexity + Why This Works
⏱ TIME: O(M × N)

Phase 1: BFS/DFS from border O's — O(M·N). Phase 2+3: scan grid twice — O(M·N). Total: O(M·N).

💾 SPACE: O(M × N)

BFS queue or DFS stack can hold up to O(M·N) cells. In-place modification — no extra grid needed.

BORDER DFS 3-PHASE REVERSE THINKING
The Reverse Insight "Find surrounded O's" is hard — you'd need to check if the entire connected region avoids the border. "Find safe O's" is easy — just BFS from all border O's. The complement is your answer.

Topological Sort

GFG · Medium — Linear ordering of vertices in a DAG such that for every directed edge u → v, vertex u appears before v. Two approaches: DFS + Stack and Kahn's Algorithm (BFS + In-degree).

🧠 Core Intuition
The Dependency Analogy Think of courses: take Maths before Physics, Physics before Chemistry. Topological sort gives the valid study order — respecting every prerequisite. Only possible in a DAG (no circular dependencies!).
DFS Approach — Push to Stack on Finish Run DFS. When a vertex has no more unvisited neighbors (fully explored), push it to a stack. The stack's pop order is the topological order — a vertex is pushed only AFTER all its descendants are pushed.
Kahn's Algorithm — BFS + In-degree Start with all vertices with in-degree 0 (no prerequisites). Process them via BFS, reducing neighbors' in-degrees. When a neighbor hits 0, enqueue it. Order of processing = topological order.
⚡ Complexity
⏱ TIME: O(V + E)

Both DFS and Kahn's visit every vertex once O(V) and every edge once O(E).

💾 SPACE: O(V)

DFS: visited array + call stack O(V). Kahn's: in-degree array + BFS queue O(V).

DAG ONLY DFS / BFS MULTIPLE VALID ORDERS
Only Works on DAGs Topological sort is undefined for graphs with cycles. Kahn's auto-detects cycles: if output has fewer than V vertices, a cycle exists.

import java.util.*; class Solution { public List<Integer> topoSort(int V, int[][] edges) { List<List<Integer>> adj = new ArrayList<>(); int[] ind = new int[V]; for (int i=0;i<V;i++) adj.add(new ArrayList<>()); for (int[] e:edges){adj.get(e[0]).add(e[1]);ind[e[1]]++;} Queue<Integer> q=new LinkedList<>(); for(int i=0;i<V;i++) if(ind[i]==0) q.add(i); List<Integer> res=new ArrayList<>(); while(!q.isEmpty()){ int u=q.poll(); res.add(u); for(int v:adj.get(u)) if(--ind[v]==0) q.add(v); } return res.size()==V?res:new ArrayList<>(); } }

Course → DAG Translation Model each course as a node. prerequisite [a,b] (take b before a) → directed edge b → a. If you can topologically sort this graph, all courses are completable. If there's a cycle → impossible.

Shortest Path in Undirected Graph (Unit Weight)

GFG · Medium — Find the shortest distance from source src to every vertex in an unweighted undirected graph. Since all edges have weight 1, BFS is the perfect tool — it naturally explores level by level, guaranteeing the first time a node is visited is via the shortest path.

🧠 Core Intuition
BFS = Shortest Path for Unit Weights BFS visits nodes in order of their distance from the source — level 0 (src), level 1 (neighbors), level 2 (neighbors of neighbors), etc. The first time BFS reaches a node is always via the shortest path, because any later path would be longer.
Distance Array Pattern Maintain a dist[] array initialized to -1 (unreachable). Set dist[src] = 0. For each neighbor v of dequeued node u: if dist[v] == -1, set dist[v] = dist[u] + 1 and enqueue.
Unreachable Vertices Stay -1 If a vertex is never visited (disconnected graph), its dist stays -1. No extra handling needed — -1 is the required output for unreachable nodes.
⚡ Complexity
⏱ TIME: O(V + E)

Each vertex is enqueued once O(V). Each edge is examined twice (undirected) O(E). Total O(V+E).

💾 SPACE: O(V + E)

Adjacency list O(V+E) + dist array O(V) + BFS queue O(V).

BFS LEVEL ORDER UNIT WEIGHT ONLY UNDIRECTED
Why Not Dijkstra? Dijkstra works for weighted graphs but is overkill here — O((V+E)logV). BFS achieves O(V+E) for unit weights. Always use BFS when all edges have equal weight.

Shortest Path in Directed Acyclic Graph

GFG · Medium — Find shortest distances from source (vertex 0) to all vertices in a weighted DAG. The key insight: Topological sort first, then relax edges in topo order. This guarantees we always process a vertex only after all paths leading to it are already finalized.

🧠 Core Intuition — Topo Sort + Relaxation
Why Topo Order Enables O(V+E) Shortest Path In a DAG, topological order guarantees that for every edge u→v, u comes before v. So when we process u, we know dist[u] is already finalized — no future edge can lead to u. Safe to relax all edges from u immediately.
Two-Phase Algorithm Phase 1: Run DFS-based topo sort → get processing order. Phase 2: Initialize dist[src]=0, rest=∞. Process vertices in topo order: for each vertex u, relax all edges u→v: if dist[u]+w < dist[v], update dist[v].
Why Not Dijkstra for DAGs? Dijkstra works but is O((V+E)logV). The topo sort approach is O(V+E) — faster! And unlike Dijkstra, it can handle negative edge weights (as long as no negative cycles, which can't exist in a DAG anyway).
⚡ Complexity
⏱ TIME: O(V + E)

Topo sort: O(V+E). Relaxation: each edge processed once O(E). Total: O(V+E).

💾 SPACE: O(V + E)

Adj list O(V+E) + dist array O(V) + topo stack O(V) + visited O(V).

TOPO SORT + RELAX WEIGHTED DAG HANDLES NEGATIVES
DAG-Only Advantage This approach is DAG-specific and beats Dijkstra on time. Can handle negative edge weights — impossible for Dijkstra. No negative cycles possible in a DAG (acyclic by definition).

def shortestPath(V, edges): # Build weighted adjacency list adj = [[] for _ in range(V)] for u, v, w in edges: adj[u].append((v, w)) # Phase 1: Topological Sort via DFS visited = [False] * V stack = [] def dfs(node): visited[node] = True for nb, _ in adj[node]: if not visited[nb]: dfs(nb) stack.append(node) # push after all neighbors for i in range(V): if not visited[i]: dfs(i) # Phase 2: Relax edges in topo order INF = float('inf') dist = [INF] * V dist[0] = 0 # src = 0 while stack: u = stack.pop() if dist[u] != INF: for v, w in adj[u]: if dist[u] + w < dist[v]: dist[v] = dist[u] + w # Replace INF with -1 for unreachable return [-1 if d == INF else d for d in dist]
import java.util.*; class Solution { public int[] shortestPath(int V, int[][] edges) { List<int[]>[] adj = new List[V]; for(int i=0;i<V;i++) adj[i]=new ArrayList<>(); for(int[] e:edges) adj[e[0]].add(new int[]{e[1],e[2]}); // Phase 1: Topo sort boolean[] vis=new boolean[V]; Deque<Integer> stack=new ArrayDeque<>(); for(int i=0;i<V;i++) if(!vis[i]) dfs(i,adj,vis,stack); // Phase 2: Relax in topo order int[] dist=new int[V]; Arrays.fill(dist,Integer.MAX_VALUE); dist[0]=0; while(!stack.isEmpty()){ int u=stack.pop(); if(dist[u]!=Integer.MAX_VALUE) for(int[] nb:adj[u]) if(dist[u]+nb[1]<dist[nb[0]]) dist[nb[0]]=dist[u]+nb[1]; } for(int i=0;i<V;i++) if(dist[i]==Integer.MAX_VALUE) dist[i]=-1; return dist; } void dfs(int u,List<int[]>[] adj,boolean[] vis,Deque<Integer> st){ vis[u]=true; for(int[] nb:adj[u]) if(!vis[nb[0]]) dfs(nb[0],adj,vis,st); st.push(u); } // Time O(V+E) Space O(V+E) }
🎯 Key Takeaways
①Topo order = safe processing order. u comes before v in topo sort means all paths to u are finalized before we relax u's edges.
②Only relax if dist[u] ≠ INF. If a vertex is unreachable, relaxing from it would write wrong distances to its neighbors.
③Handles negative weights! No cycles = no negative cycles. Topo relaxation is safe with negative edges unlike Dijkstra.
④Replace INF with -1 at end. Use INF during computation (avoids overflow). Convert to -1 only in the output.
⑤Beats Dijkstra for DAGs. O(V+E) vs O((V+E)logV). Always prefer topo sort for weighted DAG shortest path.

⚠ Bugs & Edge Cases
BUG — Relaxing from INF vertex
If dist[u] is INF (unreachable) and you relax u→v with weight w, you get INF+w which overflows or sets wrong dist[v]. Always check dist[u] != INF first.
BUG — Using -1 as INF during computation
If you use -1 as "infinity", dist[u]+w gives -1+w = wrong value. Use float('inf') or INT_MAX during relaxation, convert to -1 only at output.
EDGE — Disconnected vertices
Topo sort visits all vertices. dist stays INF for unreachable ones → converted to -1. ✓
EDGE — src with no outgoing edges
Topo sort works fine. During relaxation, src is processed from stack but adj[src] is empty. All others stay INF → -1. ✓
